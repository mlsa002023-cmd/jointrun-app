# RC1.1 Side-by-Side — 디자인 원본 vs 구현

**작성일**: 2026-07-24
**디자인 기준**: `JOINTRUN_Claude_Design_Source_Package` (HTML 5종 + PNG 23종)
**구현 기준**: `feat/v9-design-integration` (jointrun-app / jointrun-homepage)

## 0. 이 문서를 읽는 법

디자인 원본 PNG와 구현 화면을 나란히 놓고 확인한 결과다. 이 저장소에서 자동 픽셀 diff
도구는 쓰지 않았다(설치되어 있지 않음) — 대신 원본 PNG를 직접 열어보고, 같은 화면을
브라우저 390×844에서 실제로 클릭해 들어가 캡처한 뒤 눈으로 비교했다.

**분류 기준**
- **의도된 반응형 차이** — 시안에 없는 폭에서의 자연스러운 재배치. 수정 대상 아님.
- **의도된 제품 차이** — 데이터·정책·기능상 일부러 다르게 둔 것. 아래 §3에 근거를 적었다.
- **구현 오류** — 디자인 의도와 다르게 잘못 만든 것. P0/P1로 분류.
- **브라우저/폰트 차이** — 렌더링 엔진 차이. 수정 대상 아님.
- **수정 필요 없음** — 일치.

원본 PNG 파일 경로는 `screens/homepage-*.png`, `screens/app/S01~S20*.png`이며, 검수 시
같은 파일을 열어 이 표와 대조하면 된다.

맞다 — 대표님 판단이 필요한 항목이다.

## 3. 의도된 제품 차이 — 근거

### 3-1. 사진 비교 UI를 만들지 않았다
디자인의 홈페이지 DECISION LOG 섹션, S11, S15는 기준선/현재 사진을 나란히 보여준다.
그러나 `export/asset-manifest.md`가 이 이미지들을 "실사용자 촬영 이미지로 교체 필요"한
**placeholder**로 명시하고 있고, 대표님이 "대표 승인 없이 사진·영상·랜드마크 저장 기능을
추가하지 마세요 / 현재 저장되는 증상·상황·선택·결과 데이터만으로 정직한 비교 화면을
구현하세요"라고 지시하셨다. 그래서 사진 박스 대신 실제로 저장하는 증상 체감값·촬영일·
사용 손으로 비교를 구성하고, 홈페이지에는 "원본 사진은 저장하지 않습니다"를 명시했다.

### 3-2. Firestore 스키마·enum을 바꾸지 않았다
S03/S04/S07/S12/S13은 시안이 현재보다 단순한 데이터 모델을 제안한다(체감 3단계, Decision
4개 등). "현재 RC1 기능과 Firestore 구조는 다시 만들지 말고 보존하세요" 지시에 따라 필드
구조와 enum은 그대로 두고 시각 스타일만 맞췄다. 특히 S04의 손 선택(`handSide`)은 비교
가능성 판정에 쓰이는 필수 데이터라 제거하지 않았다.

### 3-3. 실제로 동작하는 기존 기능을 삭제하지 않았다
시안에 없다는 이유로 지우면 기능이 사라지는 것들: 홈페이지 리드캡처 설문 폼·파트너 섹션,
앱 홈의 레거시 스캔 대시보드, FAQ 추가 3문항. 전부 시안 기준 섹션 뒤/아래에 유지했다.

## 4. P0 / P1 / P2 정리

### P0 — 이번 스프린트에서 처리 완료 (남은 항목 없음)
- 홈페이지 헤더 내비·히어로 카피·히어로 비주얼·WHEN TO RECORD 순서 — 완료
- 홈페이지 DECISION LOG 섹션(데이터 기반으로 재해석) — 완료
- S01 온보딩 전면 교체 — 완료
- S02 개인정보·비진단 안내 신규 — 완료
- S09 홈 카운트다운 + 진행 슬라이더 — 완료
- S16 요금제 신규 — 완료

### P1 — 다음 스프린트 권장 (기능 영향 없음, 시각/동선 정렬)
1. **S06 촬영 오류 화면** — 다크 오버레이 → 밝은 배경 + 레드 CTA로 정렬
2. **S08 기준선 저장 완료** — 다크 네이비 풀스크린 + 틸 버튼으로 정렬(예정일 카드는 유지)
3. **S19 빈 상태 CTA 목적지** — "첫 스캔 시작하기"가 레거시 스캔으로 가는 것을 V9 기준선
   흐름으로 변경(현재는 V9 목적과 다른 곳으로 안내)
4. 트리거 아이콘 5종을 최종 일러스트로 교체(현재는 기하학적 placeholder)

### P2 — 여유 있을 때
1. S05 가이드 촬영 하단 카피 미세 정렬
2. S10 재확인 전용 안내 화면 분리
3. S14 타임라인 노드 색상을 디자인 3색 체계(done/current/upcoming)로 정렬
4. S13 결과 기록 pill 스타일 캡슐형 정렬
5. 여러 화면의 "뒤로" 버튼 터치 타겟 44px → 48px

### 대표님 판단이 필요한 항목
- **S18 네트워크 오류 카피** — 오프라인 큐잉을 실제로 구현할지 여부(§2 주의 참고)
- **사진 저장 정책** — 디자인의 사진 비교 UI를 실제로 쓰려면 원본 사진 저장이 필요하다.
  현재는 저장하지 않는 정책을 유지 중이며, 바꾸려면 저장 목적·최소 저장 데이터·동의 방식·
  보존 기간·삭제 방법·암호화·접근 권한을 정한 별도 승인이 필요하다.
