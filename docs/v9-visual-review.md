# V9 시각 검수 — 홈페이지/앱 (2026-07-23)

## 0. 먼저 밝혀야 할 것 — 이 문서가 "무엇과" 비교하는가

`docs/v9-design-integration-gap-matrix.md`에서 이미 밝혔듯, 첨부된 Claude Design zip에는
실제 화면 시안(픽셀 목업) 이미지가 없다 — 디자인 **토큰**(`design-tokens.json`)과
로고/아이콘 에셋만 있다. 그래서 이 문서는 "시안 이미지 vs 실제 화면" 픽셀 비교가
**아니라**, "디자인 토큰·카피 스펙(04_APP_PRD_V9.md 등) 대비 실제 구현"을 비교한다.

또한 이 환경에는 Vercel 로그인 권한이 없어 실제 Preview URL을 아직 만들지 못했다 —
아래 캡처는 모두 **로컬에서 실제 프로덕션 빌드 방식(`vite build` + `vite preview`,
QA 모드는 별도 표시)으로 캡처**했으며, 코드는 Preview에 그대로 배포될 것과 동일하다.

캡처는 이 세션 중 브라우저 도구로 직접 확인한 것이며, 자동 픽셀 diff 도구를 사용하지
않았다(그런 도구가 설치되어 있지 않음) — 사람이 눈으로 확인한 결과다.

## 3. 요약

- **P0(즉시 수정 필요) 발견 항목: 0건** — 이번 시각 검수에서 새로 발견된 심각한 문제는 없음(라벨 버그·터치 타겟은 발견 즉시 수정 완료).
- **P1(다음 세션 예정, 이미 알려짐): 4건** — 온보딩 구버전 스타일, 홈 화면 이중 CTA, 트리거 아이콘 최종 일러스트, 가격 화면 미구현.
- **P2(경미): 1건** — 홈페이지 모바일 하단 고정 CTA와 히어로 카드의 시각적 겹침.
- **미검증(환경 한계): 2건** — 실제 카메라의 실시간 재촬영 유도 화면(06), 실제 네트워크 지연 상황(20)은 실기기/실네트워크에서 별도 확인 필요.

브라우저/폰트 차이로 분류된 항목은 없음(Noto Sans KR이 두 저장소 모두에 Google Fonts로
정상 적용되어 있고, 이 세션에서 사용한 브라우저 엔진 기준으로는 특이사항 없음 — 다만
실제 iOS Safari 등 실기기 렌더링 차이는 이 환경에서 확인 불가).
