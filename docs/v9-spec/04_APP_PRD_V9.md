# JOINTRUN App PRD — V9 정렬본

## 1. 제품 목표
손가락 상태 변화가 느껴진 순간을 놓치지 않고 기록하며, 2주·4주 뒤 같은 조건으로 재확인해 과거의 나와 비교하고, 사용자의 관리 선택과 결과를 다음 판단까지 연결한다.

## 2. 비목표
- 질환 진단
- 염증·열감 자동 판정
- 변형 시점·악화 타이밍 예측
- 치료·보조기·영양제 효과 확정
- 타인 평균 또는 일반 기준으로 건강 등급 부여
- 매일 사용을 강제하는 습관 앱

## 3. 핵심 사용자
만 40~75세 중 손가락 통증·뻣뻣함·변형 우려가 있고 최근 1년 내 병원·운동·보호대·제품 등에 시간 또는 비용을 쓴 스마트폰 사용자.

## 4. 핵심 사용자 흐름
`트리거 선택 → 촬영 준비 → 가이드 촬영 → 증상·상황 기록 → 첫 기준선 저장 → 2주·4주 재확인 → 과거 비교 → 관리 선택 기록 → 결과 기록 → 다음 판단`

## 5. 정보 구조
### 하단 내비게이션
1. 홈
2. 기록하기
3. 타임라인
4. 리포트
5. 설정

### 홈의 우선순위
1. 지금 필요한 행동: 첫 기록 / 2주 재확인 / 4주 재확인 / 결과 기록
2. 다음 재확인 일정
3. 최근 Decision Loop
4. 내 타임라인

점수 대시보드, 염증 게이지, 질병 위험도는 홈에 두지 않는다.

## 6. 화면별 요구사항

### S01. 첫 진입·온보딩
**목표:** JOINTRUN이 진단 앱이 아니라 개인 기준선을 만드는 기록 서비스임을 이해시킨다.

**카피**  
제목: `오늘을 기록하면, 다음 선택의 기준이 생깁니다.`  
본문: `손가락 상태와 증상을 남기고 2주·4주 뒤 과거의 나와 비교하세요.`  
안내: `질환 진단이나 악화 예측을 제공하지 않습니다.`

**수용기준**
- 사용자가 비진단 안내를 확인할 수 있다.
- 개인정보 처리방침과 동의 화면에 접근할 수 있다.
- `시작하기` 터치 영역은 최소 48px이다.

### S02. 판단 트리거 선택
**질문:** `오늘 왜 기록하려고 하나요?`

**선택지**
- 통증·뻣뻣함이 늘어난 것 같아요
- 손가락 모양이 달라 보이는 것 같아요
- 움직임이나 손 사용이 불편해요
- 붓기·열감을 느껴요
- 병원·운동·보호대·제품을 새로 시작했어요
- 기타

**수용기준**
- 1개 이상의 트리거를 선택해야 다음으로 진행된다.
- 주 트리거 1개와 보조 트리거 복수 선택을 허용한다.
- `trigger_selected` 이벤트가 저장된다.

### S03. 촬영 전 준비
**체크 항목**
- 어느 손인지 선택
- 손 전체가 보이는지
- 밝고 고른 조명인지
- 카메라와 손의 거리가 맞는지
- 같은 배경·자세를 사용할 수 있는지

**카피**  
`이 촬영은 진단을 위한 검사가 아니라, 다음 기록과 비교하기 위한 기준을 만드는 과정입니다.`

### S04. 실시간 가이드 촬영
**기능**
- MediaPipe 21개 랜드마크로 손 위치와 프레임 정렬 확인
- 손 전체 노출 여부
- 거리·각도·흔들림·밝기 품질 체크
- 좌/우 손 일치 확인
- 실패 사유를 한 문장으로 안내

**금지 UI**
- 염증·질환 위험도
- 건강점수
- 개선/악화 자동 판정

**수용기준**
- 품질 기준 미달 시 저장보다 재촬영을 우선 제안한다.
- 품질 결과와 프로토콜 버전을 저장한다.
- `capture_started`, `capture_quality_failed`, `capture_completed`가 기록된다.

### S05. 증상·상황 기록
**입력**
- 통증 체감 0~10
- 뻣뻣함 체감 0~10
- 붓기 체감: 없음/조금/많음/모르겠음
- 열감 체감: 없음/있음/모르겠음
- 손 사용 불편: 없음/가벼움/보통/큼
- 증상이 느껴진 시점
- 오늘의 상황 메모

**주의**  
모든 값은 `사용자 체감 입력`으로 명시한다.

### S06. 첫 기준선 저장 완료
**카피**  
제목: `오늘의 상태가 첫 기준선으로 저장되었습니다.`  
본문: `2주와 4주 뒤 같은 조건으로 다시 확인하면 과거의 나와 비교할 수 있습니다.`

**기능**
- 2주·4주 재확인 예정일 표시
- 알림 동의
- 캘린더 추가 또는 앱 내 일정 생성
- 첫 기준선 카드

### S07. 홈 — 활성 기록 상태
**상태별 상단 카드**
- 첫 기록 전: `첫 기준선 만들기`
- 2주 전: `다음 재확인까지 D-n`
- 재확인 가능: `오늘 2주 기록을 다시 확인할 수 있어요`
- 4주 완료 전: `4주 비교까지 한 번 남았습니다`
- 비교 후: `선택한 관리의 결과를 기록해 주세요`

### S08. 2주·4주 재확인
**기능**
- 기준선 당시 촬영 조건 요약
- 동일 손·각도·거리·조명 안내
- 이전 사진을 반투명 가이드로 사용 가능
- 재촬영 품질 비교 가능성 판정

**수용기준**
- 예정일 전후 허용 창을 정의한다.
- 미완료·연기·취소 상태가 존재한다.
- `recheck_started`, `recheck_completed`, `recheck_skipped`가 기록된다.

### S09. 과거의 나와 비교
**표시**
- 기준선/현재 이미지 나란히 보기
- 촬영 조건 일치 여부
- 당시/현재 사용자 체감값
- 사용자가 직접 표시한 변화: 덜함/비슷함/더함/판단 어려움

**카피**  
`JOINTRUN은 관찰된 기록을 나란히 보여줍니다. 질환의 악화나 치료 효과를 판정하지 않습니다.`

**오류 상태**  
`촬영 조건 차이가 커 직접 비교가 어렵습니다. 기록은 보관되며 다음 촬영에서 다시 확인할 수 있습니다.`

### S10. Decision Log
**질문 1:** `그동안 어떤 관리를 선택했나요?`
- 운동·스트레칭
- 보호대·보조기
- 휴식·온찜질·냉찜질
- 의약품·영양제 복용
- 병원·전문가 상담
- 기타

**질문 2:** `왜 이 선택을 했나요?`
- 증상이 불편해서
- 의료진 권유
- 주변 추천
- 이전 경험
- 사용 편의성·비용
- 기타

**주의**  
앱이 치료법을 추천하는 화면이 아니라 사용자의 선택 맥락을 기록하는 화면이다.

### S11. Outcome 기록
**질문**
- 선택 후 어떻게 느꼈나요? 덜 불편함/비슷함/더 불편함/판단 어려움
- 계속할 계획인가요? 계속/변경/중단/전문가 상담/미정
- 메모

**카피**  
`이 기록은 선택과 결과의 시간적 맥락을 남기며, 인과관계를 확정하지 않습니다.`

### S12. 개인 타임라인
**카드 구조**
- 날짜·트리거
- 촬영·증상 기록
- 재확인
- 비교
- 선택
- 결과
- 다음 예정일

**필터**
- 모든 기록
- 촬영
- 선택
- 결과
- 리포트

### S13. 4주 리포트
**구성**
- 기준선과 2주·4주 기록
- 사용자 체감값 변화
- 촬영 조건 품질
- 관리 선택과 결과
- 다음 기록 권장 시점
- 비진단·비인과 안내

**금지**
- 치료 성공/실패
- 질환 악화/호전 판정
- 절대 건강점수

### S14. 상품·결제

**4주 패키지 19,900원**: 온보딩·2주/4주 알림·품질 점검·최종 비교 리포트  
**Premium 월 9,900원**: 4주 이후 장기 기록·무제한 이벤트·이전 리포트 보관

초기에는 feature flag와 예약/결제 실험을 분리한다.

- 기록 삭제
- 계정 삭제

## 7. 상태 모델
### Event 상태
`draft → capture_started → captured → baseline_created → recheck_due → rechecked → compared → decision_logged → outcome_logged → completed`

### 예외 상태
`capture_failed`, `recheck_skipped`, `comparison_unreliable`, `abandoned`, `deleted`

## 9. Definition of Done
- 핵심 경로가 모바일 360px 이상에서 끊김 없이 완료된다.
- 본문 17px 이상, 터치 영역 48px 이상, 주요 대비 WCAG AA를 충족한다.
- 점수·진단·예측·치료효과 확정 문구가 없다.
- 모든 핵심 이벤트가 분석 로그에 남는다.
- 촬영 실패·비교 불가·알림 거부·중도 이탈 상태가 처리된다.
- 사용자가 데이터 삭제와 동의 철회 경로를 찾을 수 있다.
